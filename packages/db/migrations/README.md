# Migrations

Generated by `drizzle-kit generate`. **Hand-edit the SQL** before running it in production to:

1. Add `CREATE EXTENSION IF NOT EXISTS vector;` at the top of the FIRST migration.
2. Replace `CREATE INDEX` with `CREATE INDEX CONCURRENTLY` on any table that will be large at runtime.
3. Add HNSW indexes for vector columns (Drizzle does not generate these):

```sql
CREATE INDEX CONCURRENTLY users_face_embedding_hnsw
  ON users USING hnsw (face_embedding vector_cosine_ops)
  WITH (m = 16, ef_construction = 64);

CREATE INDEX CONCURRENTLY photo_faces_embedding_hnsw
  ON photo_faces USING hnsw (embedding vector_cosine_ops)
  WITH (m = 16, ef_construction = 64);
```

## Workflow

```bash
# 1. Edit src/schema.ts
# 2. Generate migration SQL
npm run generate --workspace=@repo/db

# 3. Review and hand-edit migrations/<timestamp>_<name>.sql

# 4. Apply to local DB
npm run migrate --workspace=@repo/db
```
